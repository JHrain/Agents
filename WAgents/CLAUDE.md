- `openai` — Python SDK for LLM calls (any OpenAI-compatible API)
- `pydantic` — data models (Config, Message, ToolParameter)
- `python-dotenv` — environment variable loading (optional, used in tests)

## Architecture Overview

### LLM Layer (`core/llm.py`)
`WAgents_LLM` wraps the `openai` Python SDK to provide a unified interface to any OpenAI-compatible API. It auto-detects the LLM provider by:
1. Checking provider-specific env vars (`OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `DASHSCOPE_API_KEY`, etc.)
2. Inspecting API key format (prefix `sk-`, `ms-`, etc.)
3. Parsing `base_url` domain
4. Falling back to `LLM_API_KEY` + `LLM_BASE_URL` universal env vars

Configure via `.env` file or env vars — see `.env` for an example with DeepSeek.

### Agent Layer (`core/agent.py`, `agents/`)
`Agent` (ABC) defines the interface with `run()` + message history. `SimpleAgent` implements text-based tool calling: the LLM emits `[TOOL_CALL:tool_name:parameters]` tags, which the agent parses, executes, and feeds results back for iterative refinement.

### Tool System (`tools/base.py`, `tools/registry.py`)
Two registration paths:
- **Tool subclass**: Extend `Tool` ABC, implement `run()`, `get_parameters()`. Optionally set `expandable=True` and use `@tool_action` decorator on methods to auto-generate subtools.
- **Raw function**: Register via `ToolRegistry.register_function(name, description, func)` — simpler for stateless operations.

The `AutoGeneratedTool` class introspects method signatures and docstrings to build parameter definitions and OpenAI function-calling schemas.

### Config (`core/config.py`)
Pydantic `Config` model with env-driven defaults. Use `Config.from_env()` to load from environment.

## Key Patterns

- **Provider-agnostic LLM**: Always use the OpenAI SDK interface; set `base_url` to point at any compatible provider.
- **Text-based tool calls**: SimpleAgent uses string-parsed `[TOOL_CALL:...]` tags rather than OpenAI function calling — this keeps tool calls transparent in the text stream.
- **Expandable tools**: A `Tool` with `expandable=True` and `@tool_action`-decorated methods auto-splits into subtools at registration time.
- **Message history**: `Agent` keeps a `_history: list[Message]` that's included in each `run()` call; use `clear_history()` to reset.
